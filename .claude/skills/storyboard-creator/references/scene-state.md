# 场景状态管理

## 状态变化点标记

场景会因为剧情事件发生状态变化，必须明确标记变化点。

### 常见状态变化

| 变化事件 | 之前状态 | 之后状态 | 影响范围 |
|----------|----------|----------|----------|
| 杀人/死亡 | 环境干净 | 血迹、尸体 | 后续所有该场景分镜 |
| 打斗/破坏 | 环境整洁 | 物品散落、破损 | 后续场景 |
| 时间流逝 | 白天 | 夜晚 | 光照变化 |
| 天气变化 | 晴天 | 下雨 | 环境氛围变化 |
| 角色受伤 | 完好 | 伤口、血迹 | 该角色后续所有分镜 |

---

## 状态连贯性检查

### 检查清单

- [ ] 状态变化后的所有分镜是否都反映了新状态？
- [ ] 是否有分镜遗漏了应有的状态描述？
- [ ] 回忆/闪回场景是否与现实场景有视觉区分？
- [ ] 角色服装/伤势是否保持连贯？

### 错误示例

```
❌ 场景号 1-2-14
   镜头描述：山羊头解释游戏规则
   提示词：山羊头解释规则，众人恐惧地听着
   问题：此时桌上应该有血迹和尸体（1-1-22已杀人），但prompt没有体现
```

### 正确做法

```
✓ 场景号 1-2-14
   场景状态：杀人后
   提示词：密封混凝土房间，桌面血迹和脑浆，无头尸体瘫坐椅上，
          戴腐朽山羊头骨面具的高瘦男性站立解释规则，
          9名恐惧的中国人脸上沾血围坐聆听，昏黄钨丝灯光...
```

## 状态标记方法

### 在分镜表中添加"场景状态"列

| 场景号 | ... | 场景状态 |
|--------|-----|----------|
| 1-1-21 | ... | 杀人前 |
| 1-1-22 | ... | **杀人后**（状态变化点） |
| 1-1-23 | ... | 杀人后 |
| ... | ... | 杀人后 |

### 状态变化点高亮

在分镜表中，用粗体或特殊标记标识状态变化点，便于检查。

## 回忆场景处理

### 视觉区分

回忆/闪回场景必须与现实场景有明显视觉区分：

| 区分方式 | 描述 |
|----------|------|
| 柔和晕边 | 画面边缘模糊/暗角 |
| 色调变化 | 使用不同色温（如金色暖调） |
| 画质变化 | 略微模糊或颗粒感 |

### 提示词添加

回忆场景的提示词必须包含：`闪回柔和晕边效果`

```
✓ 闪回中景，夜晚广州高楼天台，城市灯光背景，
  两名男性对峙，地震晃动，混乱，闪回柔和晕边效果，电影感
```
